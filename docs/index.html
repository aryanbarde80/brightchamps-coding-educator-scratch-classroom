30-Minute Demo Flow

The interview asks for a live demonstration in a classroom environment. This flow keeps it structured and easy to deliver.

Introduce Explain the game goal and learning outcome in simple language.
Show Play the final output first so the learner understands the target.
Build Walk through sprites, variables, loops, events, and conditions.
Recap Review the concepts and suggest one or two extension ideas.

Architecture

The repository includes a simple visual model of how a coding classroom demo moves from idea to interaction.

Core Teaching Concepts

  • Events start the program, such as clicking the green flag.
  • Loops keep the game checking input and rules continuously.
  • Variables store changing values like score, timer, speed, and lives.
  • Conditions decide what happens when sprites touch or keys are pressed.
  • Randomness makes games feel replayable and interactive.
