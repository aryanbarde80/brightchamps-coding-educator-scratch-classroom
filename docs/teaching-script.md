# Teaching Script

## Opening

Hello everyone. Today we are going to build a small game and learn how computers make decisions. We will use simple blocks and see how each block changes the game.

## Explain The Outcome

In this project, the player controls a character or object. The goal is to collect points, avoid mistakes, and make the game respond to player actions.

## Classroom-Style Questions

- What do you think should happen when the game starts?
- Which variable should store the score?
- What should happen when the player touches the target?
- How can we make the game more difficult?
- What will happen if we increase the speed?

## Concept Recap

| Concept | Simple Explanation |
| --- | --- |
| Event | Something that starts code, like clicking the green flag |
| Loop | Repeats instructions again and again |
| Variable | Stores changing information, like score |
| Condition | Makes a decision using if/else logic |
| Collision | Detects when two objects touch |

## Closing

Today we learned how to create a game using events, loops, variables, and conditions. These concepts are used in almost every programming language, not just Scratch.
