# Interview Preparation Plan

## What You Have To Do

Prepare one simple coding project and explain it like a teacher. The interviewer is checking whether you can communicate coding concepts clearly, not whether you can build a huge app.

## Best Project To Choose

Choose **Fruit Catcher in Scratch** if you want the highest-confidence demo.

Why it works well:

- It is visual and easy to understand.
- It uses core beginner concepts.
- It can be explained in 20-30 minutes.
- It gives you natural teaching moments.

## Preparation Checklist

- Pick one project.
- Practice explaining the project objective in one minute.
- Prepare the final output before the interview.
- Practice explaining each code block.
- Keep Scratch, Python, or the browser ready before the call.
- Check internet connection and microphone.

## Suggested 4-Hour Prep Schedule

| Time | Task |
| --- | --- |
| 45 min | Understand the project and final output |
| 75 min | Build the project once from scratch |
| 45 min | Prepare the teaching explanation |
| 45 min | Practice the full demo |
| 30 min | Prepare backup project and troubleshoot |

## What To Say In The Interview

Start with:

> Today we are going to create a simple game. By the end of the class, the learner will understand events, loops, variables, and conditions.

Then explain:

- What the player does.
- What each sprite/object does.
- What variables are needed.
- What happens when the game starts.
- What happens when the player wins or loses.

## Common Interview Questions

| Question | Strong Answer |
| --- | --- |
| Why did you choose this project? | It is beginner-friendly and teaches core programming concepts visually. |
| How will you engage a child? | I will ask prediction questions and let them change values like speed or score. |
| How will you handle errors? | I will debug step by step: check events, variables, and conditions. |
| What concepts are covered? | Events, loops, variables, conditionals, coordinates, and collision detection. |
