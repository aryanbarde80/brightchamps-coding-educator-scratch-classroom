# BrightChamps Coding Educator Scratch Classroom

This repository is a ready-to-use preparation pack for the BrightChamps coding educator mock interview. It contains four Scratch classroom project plans, one HTML/CSS demo, one Python demo, teaching scripts, architecture diagrams, and the original preparation document.

## Interview Goal

The interview expects a 20-30 minute live demonstration where you teach a small coding project in a classroom style. The project can be built in Scratch, Python, or HTML/CSS.

## Recommended Demo Path

Use `scratch-projects/fruit-catcher` if you want the safest and simplest Scratch demo. It is visual, beginner-friendly, and easy to explain with events, loops, conditionals, variables, and collision detection.

**Primary selection:** Fruit Catcher in Scratch.

This selection is kept as the main interview demo because it directly matches the BrightChamps document and is the strongest option for a classroom-style explanation.

## 30-Minute Teaching Flow

1. Introduce the project and learning outcome.
2. Show the final output first.
3. Break the project into sprites, variables, and rules.
4. Build or explain the code step by step.
5. Ask the learner prediction questions.
6. Demo the finished project.
7. Recap the programming concepts.

## Quick Start

For Scratch, open [scratch.mit.edu](https://scratch.mit.edu), create a project, and follow any project guide under `scratch-projects`.

For HTML/CSS, open `html-css-project/index.html` in a browser.

For Python, run:

```bash
python python-project/number_quest.py
```
