How To Present The Demo

The interview is not only about code. It checks whether the project can be explained like a teacher in a classroom.

1. Introduce Tell the learner they will build a simple Fruit Catcher game.
2. Show Output Show basket movement, falling fruit, and score before explaining code.
3. Explain Sprites Break the project into basket, fruit, backdrop, variables, and rules.
4. Teach Logic Explain green flag, forever loop, keyboard input, random x, and touching.
5. Recap Review events, loops, variables, conditions, and collision detection.
